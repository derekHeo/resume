# 하나증권 디지털(AI) 자기소개서 전략

## 인재상 키워드 매핑

| 가치 | 핵심 | 내 경험에서의 연결고리 |
|---|---|---|
| 온기 | 고객 가치 창출, 따뜻한 마음 | 파라나AI(고령 사용자), 갈등관리(비개발직군 소통) |
| 호기 | 변화·혁신, 도전정신, 미래 창조 | AI 취업 어시스턴트(0→1 풀스택), 금융 대회 연속 도전 |
| 동기 | 열정·협력, 함께 성장 | What If 팀프로젝트 → ACM CHI, 인턴→선임연구원 |

## 전체 스토리 흐름 (5문항 관통 서사)

> **금융 데이터에 대한 호기심(1번)**
> → 그 호기심을 실력으로 전환한 과정(2번)
> → 기술적 난관을 돌파한 끈기(3번)
> → 다양한 사람과 함께 성과를 만든 소통력(4번)
> → 이 모든 것이 하나증권 디지털(AI)팀에서 어떻게 쓰이는가(5번)

## 1번. 지원 이유 + 금융투자업 선택 계기 (최소 1000자)

### 인재상 연결: 호기 (미래에 대한 도전)

### 권장 키워드
- 금융 데이터, KRX 경진대회, NH 빅데이터 대회
- 데이터가 의사결정을 바꾸는 경험
- 하나증권 디지털(AI)팀의 방향성
- 에이전트, 데이터 파이프라인

### 서사 구조
1. **도입 — 금융 데이터와의 첫 접점**
   - KRX 주식 투자 알고리즘 경진대회 참가 (2023)
   - 주가 데이터를 직접 다루면서 "금융 데이터가 실제 투자 의사결정에 영향을 준다"는 점에 매력을 느낀 계기
   - 단순 예측이 아니라, 데이터 구조화 → 전략 설계 → 실행까지의 파이프라인에 흥미

2. **발전 — 금융 도메인 경험 확장**
   - NH 투자증권 빅데이터 경진대회 (2024)로 이어진 지속적 관심
   - 금융 데이터 분석이 고객 경험과 비즈니스 인사이트에 직결된다는 확신

3. **왜 하나증권인가**
   - 하나증권의 디지털(AI)팀이 에이전트 설계·데이터 파이프라인·고객 세그멘테이션 등 "데이터로 금융 서비스의 미래를 만드는" 조직이라는 점
   - 생성형 AI 기술을 금융에 접목하는 과정에 기여하고 싶다는 포부
   - "미래에 대한 호기" — 변화를 두려워하지 않고 새로운 기술 도메인에 뛰어드는 자세와 부합

### 주의
- "금융에 관심이 있습니다" 같은 막연한 표현 X
- 실제 대회 참가 → 무엇을 느꼈는지 구체적 경험 기반으로

## 2번. 직무 준비 경험과 노력 (최소 1000자)

### 인재상 연결: 호기 + 동기 (도전 + 성장)

### 권장 키워드
- 데이터 추출/처리/정제, 파이프라인 설계
- 생성형 AI 아키텍처 (RAG, LLM, CoT)
- 시스템 연계 개발 (크롤러 → LLM → Auto-Fill)
- SQL, Python, Pandas, scikit-learn
- 논리적 데이터 구조화

### 서사 구조
1. **데이터 파이프라인 역량 — AI 취업 어시스턴트**
   - 채용 공고 크롤링(Playwright+BS4) → LLM 파싱 → 적합도 분석 → 자소서 생성 → 크롬 확장 Auto-Fill
   - end-to-end 데이터 파이프라인을 혼자 설계·구축한 경험
   - FastAPI 백엔드 계층 분리 (Router → Service → Repository), AWS 인프라, CI/CD
   - → 직무 요구사항 "데이터 파이프라인 및 시스템 연계 개발"에 직결

2. **생성형 AI 아키텍처 이해 — RAG 챗봇**
   - LangChain 기반 RAG 파이프라인 설계·운영
   - 키워드 필터 → CoT 목적 분류 2단계 라우팅 구조
   - → 직무 요구사항 "에이전트 설계" + "생성형 AI 기술/아키텍처 이해도"

3. **데이터 분석·정제 역량 — 딥러닝 분류모델 & 금융 대회**
   - 수도센터 5년치 민원 데이터 분석·가공·라벨링 → 학습 데이터 구축
   - TF-IDF, Sentiment Analysis, Word2Vec 등 텍스트 데이터 분석
   - BERT-LSTM 모델 성능 3% 개선
   - → 직무 요구사항 "데이터 정제/최적화", "논리적 데이터 구조화"

4. **기술 스택 정리**
   - Python, SQL, Pandas, scikit-learn, PyTorch, LangChain, Docker, AWS
   - ADsP, DAsP, SQLD 자격증 → 데이터 기본기 증명

### 주의
- 나열식 X → 각 경험이 직무의 어떤 요구사항에 매핑되는지 명시
- "~했습니다"로 끝내지 말고 "이 경험을 통해 ~역량을 갖추게 되었습니다"로 연결

## 3번. 목표 달성 위해 어려움 극복 (최소 500자)

### 인재상 연결: 호기 (도전정신으로 난관 돌파)

### 권장 키워드
- 기술적 병목, 구조 개선, 성능 최적화
- 문제 정의 → 원인 분석 → 해결 → 결과(숫자)

### 추천 소재: RAG 챗봇의 2가지 기술 난관

### 서사 구조
1. **목표**: 군부대원 대상 갈등관리 RAG 챗봇을 실사용 가능한 수준으로 완성
2. **어려움 1 — 불필요한 RAG 호출로 인한 응답 지연 + 비용 증가**
   - 모든 대화에 RAG를 태우는 구조 → 응답 느림, API 비용 폭증
   - "키워드 필터 → CoT 목적 분류" 2단계 라우팅 설계로 해결
   - RAG가 필요한 질의만 선별 → 불필요한 호출 제거
3. **어려움 2 — GCP Cold Start로 응답 10초**
   - Python 단일 구조의 한계 파악
   - JS 혼합 구조로 전환 → 응답 시간 10초 → 5초로 단축
4. **결과 및 배운 점**
   - 산학 과제 성공적 완수
   - "문제를 정의하고 구조를 바꾸는 것이 진짜 최적화"라는 교훈

### 주의
- 감정 서사("힘들었지만 포기하지 않고") 최소화
- 문제 → 원인 분석 → 해결책 → 수치 결과 중심으로

## 5번. 강점이 하나증권에 기여하는 방법 (최소 500자)

### 인재상 연결: 온기 + 호기 + 동기 (3가지 통합)

### 권장 키워드
- 데이터 파이프라인, 생성형 AI 에이전트
- 고객 세그멘테이션, 비즈니스 인사이트
- 도메인 적응력, 기술 커뮤니케이션
- 금융 데이터, 고객 가치

### 서사 구조
1. **강점 1: end-to-end 데이터 파이프라인 설계 능력**
   - AI 취업 어시스턴트에서 크롤링→분석→생성→서빙 전체 파이프라인 구축 경험
   - → 하나증권의 데이터 파이프라인 및 시스템 연계 개발에 바로 기여
   - → 에이전트 데이터 분석/설계/정제/최적화 업무 수행 가능

2. **강점 2: 생성형 AI 아키텍처에 대한 실전 이해**
   - RAG, CoT, LLM 파이프라인을 실서비스 수준으로 구현·최적화한 경험
   - → AI 에이전트 설계·고도화에 기여

3. **강점 3: 도메인 빠른 흡수 + 기술 커뮤니케이션**
   - 갈등관리(심리), 수도센터(공공), 금융(대회) 등 다양한 도메인 경험
   - 비개발 직군과의 협업 경험 → 금융 현업(트레이더, RM, 마케팅)과 소통하며 비즈니스 인사이트 도출 가능
   - → "사람에 대한 온기" — 기술자이되 고객과 현업의 언어로 소통

4. **마무리**: 세 가지 강점을 바탕으로 하나증권 디지털(AI)팀에서 데이터로 고객 가치를 만들어내는 데 기여하겠다
