  공고 맞춤 보강 포인트 5가지

  1. Toss ML Challenge 프로젝트를 "광고 추천팀" 관점으로 살짝 재프레이밍

  현재 서술이 "모델을 만들었다"에 가까운데, 당근 광고추천팀이 하는 일과 연결하면 좋습니다.

  현재: "사용자의 광고 클릭 확률을 예측하는 모델 개발"

  보강안: "사용자 행동 시퀀스와 광고 피처를 결합하여 클릭 확률을 예측하는 CTR 모델 개발. 클릭률 약
  2%의 클래스 불균형 환경에서 데이터 기반 피처 선택과 하이브리드 아키텍처로 예측 정밀도를 개선"

  포인트: "사용자 행동 시퀀스"라는 표현이 들어가면 당근이 하는 개인화 추천과 자연스럽게 연결됩니다.
  실제로 seq 피처가 사용자 행동 시퀀스니까 거짓말도 아니고요.

  2. "가설 → 검증" 사고 과정을 드러내기

  공고에서 반복적으로 강조하는 패턴이 **"데이터 분석 → 가설 → 실험 검증 → 임팩트"**입니다. Toss ML
  Challenge의 What did I do에 이 흐름을 녹일 수 있어요.

  예를 들어 현재:
  "37개 히스토리 피처를 PCA로 차원 축소"

  이걸 가설-검증 구조로 바꾸면:
  "히스토리 피처 간 높은 상관관계를 확인하고, PCA 차원 축소가 노이즈 제거에 유효하다는 가설 하에
  적용"

  과하게 늘리진 않더라도, "왜 이걸 했는지"가 한 마디라도 들어가면 인상이 달라집니다.

  3. What If 프로젝트의 "실험 설계" 측면 강조

  당근이 원하는 **"온라인 실험으로 검증하고 실질적인 서비스 임팩트"**와 가장 가까운 경험이 What
  If입니다.

  현재 Description:
  "2주간 실험 결과 사용량 21% 감소 검증"

  보강안:
  "피실험자 8명 대상 2주간 대조군/실험군 비교 실험을 설계하고, 스마트폰 사용량 21% 감소라는
  통계적으로 유의미한 결과를 도출"

  **"대조군/실험군 비교 실험 설계"**라는 표현이 A/B 테스트 경험과 간접적으로 연결됩니다.

  4. 자기소개 문구 보강

  현재:
  "소통을 매우 중요하게 생각하며... ML / LLM / MLOps 에 관심이 있습니다."

  이건 너무 일반적입니다. 공고에서 "문제를 체계적으로 정의하고 합리적 근거 기반 해결"을 강조하는데,
  자기소개에서 이 톤을 잡아주면 좋습니다.

  보강안 예시:
  "데이터에서 문제를 정의하고, 가설 기반으로 모델을 설계·검증하는 과정을 즐깁니다. 추천 시스템과 CTR
   예측 등 사용자 행동 기반 ML에 관심이 있습니다."

  5. Skills에 추가 고려할 것들

  현재 Skills에 없지만 실제로 사용한 것:
  - LightGBM — CTR 대회에서 피처 선택에 사용. 당근도 추천/광고에서 GBDT 계열을 쓸 가능성 높음
  - scikit-learn — StandardScaler, train_test_split 등 실제 사용

  ---
  요약: 이력서 전체 톤 조정 방향

  당근이 찾는 사람은 **"모델을 잘 만드는 사람"이 아니라 "문제를 정의하고 실험으로 검증해서 서비스
  임팩트를 만드는 사람"**입니다.

  현재 이력서는 "무엇을 했다(What)" 중심인데, **"왜 했고 어떤 판단을 했다(Why)"**를 한 줄씩만
  추가해도 공고와의 핏이 크게 올라갑니다. 실제 반영할 때 알려주시면 수정 도와드리겠습니다.
